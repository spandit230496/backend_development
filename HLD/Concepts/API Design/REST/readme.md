REST (Representational State Transfer)

REST is an architectural style that uses HTTP/HTTPS. Data is exchanged in JSON or XML. It's stateless, meaning each request is independent. It's simple, widely supported, and works well with browsers and mobile apps.
Use REST when:

Building public APIs consumed by browsers or mobile apps
Simplicity and developer friendliness matter
You need broad tooling and ecosystem support
Caching is important (REST responses are naturally cacheable)
