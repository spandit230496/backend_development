SOAP (Simple Object Access Protocol)

SOAP is a protocol (not just a style) that uses XML exclusively. It has strict standards, built-in error handling, and supports WS-Security, making it enterprise-grade. It's heavyweight and verbose compared to REST and gRPC.
Use SOAP when:

Working in enterprise environments (banking, insurance, government)
You need built-in security standards (WS-Security)
Formal contracts and strict compliance are required
Legacy systems already use SOAP
